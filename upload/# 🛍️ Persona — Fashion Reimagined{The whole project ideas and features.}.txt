# 🛍️ Persona — Fashion Reimagined

## Complete Project Specification

---

## 1. Project Overview

**Persona** is an AI-powered, multi-department fashion e-commerce platform that reimagines the online shopping experience. It combines cutting-edge technology with luxury aesthetics to deliver a personalized, immersive shopping journey across Women's, Men's, Kids', and Accessories departments.

**Brand Identity:**
- **Name:** Persona — Fashion Reimagined
- **Tagline:** "Your Style. Your Story. Your Persona."
- **Color Palette:**
  - Gold: `#C9A96E` (luxury accent)
  - Rose: `#E8A0BF` (feminine touch)
  - Dark: `#0F0F0F` (premium backgrounds)
  - Cream: `#FAF8F5` (light backgrounds)
- **Design Language:** Glassmorphism + Bento-box layouts + Micro-animations

---

## 2. Core Features

### 2.1 🏠 Homepage Experience

| Feature | Description |
|---------|-------------|
| **Hero Carousel** | Full-width image carousel with auto-play, manual navigation, and CTA buttons. Each slide showcases a department or seasonal campaign. Framer Motion parallax effects on scroll. |
| **Category Grid** | Bento-style grid of department categories (Women, Men, Kids, Accessories, Shoes, Beauty, Sportswear, Luxury). Each card has a hover-zoom image effect, overlay text, and link to department. Responsive: 4 columns → 2 columns on mobile. |
| **Featured Products** | Horizontally scrollable product carousel highlighting curated/featured items. Product cards show image, name, price, rating, and quick-add button. |
| **Flash Deals Section** | Time-limited deals with live countdown timers. Each deal card shows original price, sale price, discount percentage, and time remaining. Urgency-driven design with pulsing "HOT" badges. |
| **New Arrivals** | Grid of latest products sorted by date. "NEW" badge overlay. Lazy-loaded images with skeleton placeholders. |
| **Best Sellers** | Top-selling products ranked by sales volume. Trophy/fire emoji badges for top 3. |
| **Brand Marquee** | Infinite horizontal scroll of partner brand logos. Auto-scroll with pause on hover. |
| **Promo Banner** | Full-width promotional banner (mid-page) for seasonal sales, limited collections, or brand collaborations. |
| **Newsletter Signup** | Email input with animated subscribe button. Success/error toast notifications. |
| **Department Hub** | AI-powered department showcase — each department has its own visual identity, AI style recommendations, and curated collections. |

### 2.2 🔍 Product System

| Feature | Description |
|---------|-------------|
| **Product Card** | Image with hover-zoom, product name, brand, price (with sale display), star rating, wishlist heart icon, quick-add to cart. |
| **Product Grid** | Responsive grid (4→3→2→1 columns). Filter sidebar for size, color, price range, brand. Sort by: Newest, Price Low-High, Price High-Low, Best Selling, Rating. |
| **Product Quick View** | Dialog/modal showing product details without leaving the page. Image gallery, size selector, color selector, add to cart. |
| **Product Image Gallery** | Multi-image carousel with thumbnail navigation. Zoom on hover. |
| **Product Price** | Displays original price (strikethrough if on sale), sale price, and percentage discount badge. Supports multi-currency display. |
| **Product Rating** | Star rating (1-5) with review count. Hover to see rating breakdown. |
| **Product Badges** | Smart badges: SALE, NEW, HOT, LIMITED, EXCLUSIVE, LOW STOCK. Color-coded and animated. |
| **Add to Cart** | Animated button with quantity selector (+/-). Size/color selection required before adding. Cart icon bounce animation on add. |
| **Wishlist** | Heart icon toggle with fill animation. Wishlist stored in localStorage with Zustand persistence. |

### 2.3 🛒 Cart System

| Feature | Description |
|---------|-------------|
| **Cart Sheet** | Slide-out panel (from right in LTR, from left in RTL) using Shadcn Sheet component. Shows all cart items, quantities, and totals. |
| **Cart Item** | Product image thumbnail, name, size, color, quantity controls (+/-), price, remove button. Swipe-to-delete on mobile. |
| **Cart Summary** | Subtotal, shipping estimate, tax, discount code input, total, and "Proceed to Checkout" button. |
| **Cart Count Badge** | Animated badge on header cart icon showing item count. Bounces on update. |
| **Persistent Cart** | Zustand store with localStorage persistence. Cart survives page refresh and browser restarts. |
| **Cart API** | REST API endpoints for add, update quantity, remove item, clear cart. Syncs with database for authenticated users. |

### 2.4 📂 Category System

| Feature | Description |
|---------|-------------|
| **Category Card** | Full-bleed image with gradient overlay, category name, item count. Hover: image zoom + text shift animation. |
| **Category Navigation** | Horizontal scrollable nav bar below header. Active category highlight. Smooth scroll to section. |
| **Sub-categories** | Dropdown/drawer showing sub-categories within each department (e.g., Women → Dresses, Tops, Jeans, Activewear). |
| **Category Breadcrumbs** | Breadcrumb navigation for deep category browsing. |

### 2.5 ⚡ Deals System

| Feature | Description |
|---------|-------------|
| **Deal Card** | Product image, sale price, original price, discount percentage badge, countdown timer. |
| **Countdown Timer** | Real-time countdown (Days, Hours, Minutes, Seconds). Color changes as deadline approaches (green → yellow → red). |
| **Deal Banner** | Full-width promotional banner with countdown, CTA button, and deal description. |
| **Deal Progress Bar** | Shows percentage of stock sold (e.g., "78% sold") creating urgency. |

### 2.6 🌍 Internationalization (i18n)

| Feature | Description |
|---------|-------------|
| **Multi-locale Routing** | URL-based locale detection: `/en/...` for English, `/ar/...` for Arabic. |
| **Language Switcher** | Toggle in header between 🇬🇧 EN and 🇸🇦 AR. Animated transition between languages. |
| **RTL Support** | Full right-to-left layout when Arabic is selected. Mirrored navigation, reversed flex/grid direction, aligned text. |
| **Complete Translations** | Every UI string translated to Arabic — zero English leakage in Arabic mode. Includes product names, category labels, button text, error messages, placeholders, etc. |
| **Locale Detection** | Auto-detect browser language on first visit. Store preference in cookie. |
| **Number Formatting** | Arabic numeral formatting for prices and quantities in Arabic locale. |
| **Date Formatting** | Arabic date format for deal countdowns and product dates. |
| **Middleware** | Next.js middleware handles locale prefix, redirects, and cookie management. |

### 2.7 🎨 Theme System

| Feature | Description |
|---------|-------------|
| **Light/Dark Mode** | Toggle between light (cream background) and dark (dark background) themes. Smooth CSS transition. |
| **Theme Persistence** | Theme preference stored in localStorage and cookie. Respects system preference on first visit. |
| **CSS Variables** | All colors defined as CSS custom properties for instant theme switching. |

### 2.8 📱 Responsive Design

| Breakpoint | Layout |
|------------|--------|
| **Mobile** (< 640px) | Single column, hamburger menu, bottom sticky cart bar, swipe gestures |
| **Tablet** (640px - 1024px) | 2-column grid, collapsible sidebar, touch-optimized |
| **Desktop** (1024px+) | 4-column grid, full header with search, hover effects, keyboard navigation |

### 2.9 ♿ Accessibility (WCAG 2.1 AA)

| Feature | Description |
|---------|-------------|
| **Semantic HTML** | Proper heading hierarchy, landmark regions (`main`, `nav`, `header`, `footer`) |
| **ARIA Labels** | All interactive elements have descriptive aria-labels |
| **Keyboard Navigation** | Tab order, focus trapping in modals, skip-to-content link |
| **Screen Reader** | sr-only text for icon-only buttons, live regions for cart updates |
| **Color Contrast** | All text meets 4.5:1 contrast ratio minimum |
| **Touch Targets** | Minimum 44px × 44px for all interactive elements |
| **Motion Reduction** | Respects `prefers-reduced-motion` media query |

## 4. Data Models

### 4.1 Prisma Schema

```prisma
model Product {
  id          String   @id @default(cuid())
  name        String
  nameAr      String?           // Arabic name
  description String
  descriptionAr String?         // Arabic description
  price       Float
  salePrice   Float?            // Nullable = not on sale
  brand       String
  images      String            // Comma-separated image URLs
  category    String            // women, men, kids, accessories, shoes, beauty
  subcategory String?           // dresses, tops, jeans, etc.
  sizes       String            // Comma-separated: XS,S,M,L,XL,XXL
  colors      String            // Comma-separated: Black,White,Red
  rating      Float    @default(0)
  reviewCount Int      @default(0)
  inStock     Boolean  @default(true)
  stockCount  Int      @default(100)
  isFeatured  Boolean  @default(false)
  isNew       Boolean  @default(false)
  isBestSeller Boolean @default(false)
  tags        String?           // Comma-separated tags
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
}

model Category {
  id          String  @id @default(cuid())
  name        String
  nameAr      String?           // Arabic name
  slug        String  @unique
  image       String            // Category image URL
  description String?
  descriptionAr String?
  order       Int     @default(0)
  isActive    Boolean @default(true)
}

model Deal {
  id          String   @id @default(cuid())
  productId   String
  product     Product  @relation(fields: [productId], references: [id])
  title       String
  titleAr     String?           // Arabic title
  discountPercent Int
  startsAt    DateTime
  endsAt      DateTime
  isActive    Boolean  @default(true)
  maxQuantity Int?              // Max sale quantity
  soldCount   Int      @default(0)
  createdAt   DateTime @default(now())
}

model CartItem {
  id          String  @id @default(cuid())
  userId      String?           // Null for guest carts
  sessionId   String?           // For guest identification
  productId   String
  product     Product @relation(fields: [productId], references: [id])
  quantity    Int     @default(1)
  size        String?
  color       String?
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
}
```

## 5. API Routes

| Method | Route | Description |
|--------|-------|-------------|
| `GET` | `/api/products` | List products with filters (category, search, sort, page) |
| `GET` | `/api/products?featured=true` | Get featured products |
| `GET` | `/api/products?new=true` | Get new arrivals |
| `GET` | `/api/products?bestSeller=true` | Get best sellers |
| `GET` | `/api/categories` | List all categories |
| `GET` | `/api/deals` | List active deals |
| `GET` | `/api/cart` | Get current user's cart |
| `POST` | `/api/cart` | Add item to cart |
| `PATCH` | `/api/cart` | Update cart item quantity |
| `DELETE` | `/api/cart` | Remove item from cart |
| `GET` | `/api/auth/[...nextauth]` | NextAuth.js authentication |

## 8. Performance Targets

| Metric | Target |
|--------|--------|
| First Contentful Paint | < 1.5s |
| Largest Contentful Paint | < 2.5s |
| Time to Interactive | < 3.5s |
| Cumulative Layout Shift | < 0.1 |
| Lighthouse Score | > 90 |

### Performance Strategies
- **Image Optimization:** Next.js `<Image>` with WebP, lazy loading, blur placeholders
- **Code Splitting:** Dynamic imports for heavy components (Cart, QuickView)
- **Caching:** Local memory caching for API responses, ISR for product pages
- **Bundle Size:** Tree-shaking, only import needed Radix primitives
- **Skeleton Loading:** Instant feedback with shimmer placeholders
